Git Rules for Deliverable #1 — Creational Design Patterns
======================================================

1. Main branches: main (protected), develop (integration)
2. No direct commits to main or develop
3. Feature branches:
   - Andrew: feature/andrew-singleton-factory
   - Adham: feature/adham-abstract-factory
   - Yassin: feature/yassin-qa-packaging
4. PR required + 1 reviewer minimum
5. Only merge if NetBeans Clean and Build + Run PASS
6. Merge policy: merge commits (no rebase) for simplicity

Enforcement:
- Any PR violating these rules will be rejected and must be fixed before re-submission.
- These rules apply to all team members (Adham, Andrew, Yassin).
